# Scenario Stories

This folder is reserved for Storybook runtime validation of real product states and data pressure tests, following `docs/storybook-classification.md`.

## Belongs Here

- NoData, LowProgress, GoalCompleted, Overachieved, DecliningTrend, FirstUse, PermissionDenied, and LongHistory scenario validation
- Cross-component state pressure tests
- Multilingual and long-content checks
- Scenario reviews linked to real evidence after source mapping exists

## Does Not Belong Here

- Component variants that do not describe a user situation
- Pure style-system previews
- Fake data presented as source evidence
- Experiments treated as stable rules
- New product directions outside the approved project scope

No story files are created in this preparation step.
