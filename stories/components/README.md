# Component Stories

This folder is reserved for Storybook runtime validation of reusable semantic component candidates, following `docs/storybook-classification.md`.

## Belongs Here

- Component state validation for candidates documented in `specs/component-anatomy.md`
- Default, loading, empty, error, long-text, and extreme-value states
- Accessibility and data edge-case validation
- Evidence-linked component reviews after real sources are mapped

## Does Not Belong Here

- Page-level pattern composition
- Unreviewed experiments
- Components promoted only because they appear in a source file
- Fabricated Figma node references
- UI implementation work before the evidence and registry mapping layer is ready

No story files are created in this preparation step.
