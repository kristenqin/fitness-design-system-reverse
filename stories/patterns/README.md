# Pattern Stories

This folder is reserved for Storybook runtime validation of page-level and section-level composition patterns, following `docs/storybook-classification.md`.

## Belongs Here

- Pattern validation for candidates documented in `specs/page-patterns.md`
- Composed surfaces that test information hierarchy
- Default and edge-state pattern reviews
- Evidence-linked pattern scenarios after real sources are mapped

## Does Not Belong Here

- Isolated component-only stories
- Style-only token previews
- Unstable migration experiments
- Product claims without source evidence
- New patterns outside the current first-cycle boundary

No story files are created in this preparation step.
